# Operational Runbooks

This folder contains the minimum operational package for production support.

## Contents

- `oncall-checklist.md`: Shift-start and shift-end checklist for responders.
- `incident-response.md`: Severity model, response process, and communications.
- `alert-thresholds.md`: Baseline alert conditions and initial thresholds.

## Ownership

- Primary owner: Engineering (application reliability)
- Secondary owner: Operations (incident coordination)

## Review Cadence

- Review monthly and after every Sev-1/Sev-2 incident.
- Update immediately when architecture, auth model, or critical dependencies change.
